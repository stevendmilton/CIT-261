Local Storage

Web Storage API

In the past, you had a couple ways you could store data from one page to the next. You could either pass the data in the actual web page call as is done in PHP where the data is pass as tokens in the web address or you could write the data to a cookie that could be accessed from any page. With the advent of HTML5, a specification call Web Storage was created that allows you to create local storage on your client that can be accessed from one page to the next making HTML that much more powerful and useful.

There are two types of Web Storage. There is localStorage which keeps the variable and its data indefinitely until destroyed and there is sessionStorage which disappears when the browser session is shutdown. You will have to decide when each type should be used and when it makes sense to keep the data on the client past closing out the session. HTML5 storage is base on named key/value pairs which must be a string. This means you are unable to store dates or functions. Let's look at how to use this storage.

Local Storage Example

localStorage.setItem("person", {name:"Steven", age:56, employed:false, interests: ["golf","reading", "tennis"]});

This will create a local variable called person with the JSON information for that person's definition. Once this variable has been stored, we can then retrieve it as follows:

var person = localStorage.getItem("person");

If you want to remove the localStorage item, you would use the method removeItem as follows:

localStorage.removeItem("person");

When the link is clicked, it will open a new page which will then get the localStorage JSON object with a string, number, boolean, and array value which will then be parsed then displayed to the screen.
